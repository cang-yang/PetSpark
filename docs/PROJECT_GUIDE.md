# PetSpark 项目指南

## 1. 产品定位

PetSpark（派宠）面向宠物家庭、平台运营人员和内容审核人员，目标是把分散的宠物资料、服务预约、健康信息、领养救助、社区消费与 AI 陪伴连接为一条连续的宠物生活管理链路。

平台不是在线诊断工具。健康记录用于整理事实，AI 用于提供一般性信息与就医边界提示，医疗决策必须由专业人员完成。

## 2. 用户与权限

| 角色 | 主要能力 |
|---|---|
| 游客 | 浏览公开首页、宠物、商品、社区与服务信息 |
| 注册用户 | 管理个人资料和宠物、预约服务、创建订单、参与社区、使用 AI |
| 运营人员 | 处理商品、服务、内容、救助线索和运营横幅 |
| 服务人员 | 履约寄养、训练、美容或医疗预约 |
| 审计人员 | 查看允许范围内的审计信息 |
| 平台管理员 | 用户、角色、系统配置与全局业务管理 |

前端只展示用户拥有权限的入口，后端仍对每个受保护接口执行鉴权与所有权检查。

## 3. 主要业务链路

### 宠物档案与健康

用户创建或绑定宠物档案，维护基础资料和图片，并按时间记录疫苗、体检、用药与护理变化。涉及他人宠物时，接口根据所有权和公开状态限制访问。

### 服务与订单

用户浏览服务项目并创建预约；管理端负责资源、时段和履约状态。商城订单与各类服务预约拥有独立状态机，状态变化会产生站内通知和审计记录。

### 领养与救助

公开宠物可以进入领养流程，申请、审核与交接均保留状态记录。流浪宠物线索由用户提交，运营人员核实、处理和关闭。

### AI 能力

- 宠物对话：讯飞星火原生 SSE，模型生成一段即转发一段。
- 护理问答：结构化输出、安全规则和高风险求助提示。
- 智能推荐：只允许推荐请求时平台中真实、可见的候选对象。

AI 输入先脱敏并检查提示词注入；输出经过边界处理后加密存储。调用记录只保存必要元数据与输入哈希。

## 4. 代码结构

后端按业务模块组织 Controller、Service、Repository 和 DTO；公共 API 响应、错误、日志、安全及审计能力集中在公共模块。数据库结构全部由 Flyway 迁移维护。

前端由路由模块、API 客户端、用户端页面、管理端页面和基础 UI 组件组成。认证信息由 Vuex 管理，HTTP 层统一处理错误与未授权跳转。

## 5. 配置原则

- 开发环境使用被 Git 忽略的 `.env.local` 与 `.env.spark.local`。
- 生产环境使用 VPS `/opt/petspark/.env` 和 GitHub Actions Secrets。
- 示例文件只包含字段名和安全占位符。
- 密钥一旦用于加密生产数据，不得随意替换；更换必须设计迁移过程。

## 6. 数据库与演示数据

应用启动时由 Flyway 按版本顺序迁移数据库。演示初始化器使用稳定标识和幂等写入，可在空生产库生成完整展示内容，不需要复制开发数据库，也不会删除用户自行创建的数据。

生产数据库和上传目录应由 1Panel 或其他备份系统定期备份；自动发布前还会生成一次数据库快照并保留最近版本。

## 7. 验证与发布

每个 PR 必须通过后端测试、前端测试、生产构建和 Docker 镜像构建。合并到 `main` 后，CD 构建不可变 SHA 镜像，通过 SSH 部署并执行健康检查；失败时恢复上一成功镜像。

详细命令见仓库 [README](../README.md)，生产操作见 [部署手册](../deploy/README.md)。

## 8. 安全与隐私边界

- 密码仅保存 BCrypt 哈希。
- JWT 有效期、刷新、撤销和 token version 共同控制会话。
- 手机号、AI 消息等敏感内容加密存储。
- 上传文件校验类型、大小、文件名和路径。
- 日志不记录密码、Token、SMTP 授权码或完整敏感信息。
- 健康、位置、联系方式和图片只在完成功能所需范围内处理。
